Stop Management & Creation (Live Updates)

Add New Stop Details

Define a new stop. The Stop ID will be auto-suggested based on format (inferred or randomly chosen based on name) and existing stop patterns. You can edit the suggested ID. A placeholder schedule entry will be added to make it available in selectors. The Stop ID must be unique. Changes are saved directly to Firebase.

Bulk Merge or Add Stops (via Excel File)

This tool will MERGE data from the uploaded Excel file with your existing stops and schedules.
- If a StopID is provided in Excel and matches an existing stop, that stop's Name and Direction will be updated. New LineName(s) from Excel will be added as placeholder schedule entries if they don't already exist for that stop.
- If a StopID is provided but does not exist, it will be added as a new stop (ID must be unique).
- If StopID is blank, the system tries to match by StopName's base. If matched, details are updated. If not matched, a new stop with an auto-generated ID is created.
- Existing stops and schedule entries NOT mentioned in the Excel file will remain untouched. This tool primarily adds new information or updates existing stop details.
Expected Excel columns:
Column A: StopID (Optional - if blank, logic above applies. Must be unique if creating a new stop with a provided ID.)
Column B: StopName (Required for new stops or if StopID is blank)
Column C: Direction (Optional - "Towards" text)
Column D: LineName(s) (Optional - comma-separated, e.g., "10A,12,X26". Defaults to "INFO_ONLY" for new stops if blank, or if adding new lines to existing stops.)
The first row of the sheet should contain data, not headers. All changes are saved directly to Firebase.
